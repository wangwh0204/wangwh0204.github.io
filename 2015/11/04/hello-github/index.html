这是我在github上的第一篇blog，本blog的内容即描述如何在github上完成本篇blog. 所有内容基于如下知识和工具构建:

git

git是一个分布式版本控制系统(VCS)，通过完整的阅读一遍pro git这本书可以熟悉git及github的概念和用法.

github

github是一个基于git软件开发社区.阅读got github进一步熟悉github

markdown

markdown是一个简单的基于文本的标记语言，我主要使用markdown和github flavored markdown标记来写文章

hexo

虽然github官方推荐使用jekell来做blog内容，但我对ruby没兴趣，正好最近在了解JavaScript，所以采用基于node.js的hexo来维护blog内容。

do it in Linux

安装git, nodejs, hexo

参考https://hexo.io/docs/

在github创建blog仓库

在github创建新仓库 wangwh0204.github.io，这个仓库就可以直接作为个人blog站点的内容。通过 http://wangwh0204.github.io 即可访问，内容由hexo生成。

使用hexo

  • 配置_config.yml

  • 选择 theme

  • 预览

blog源文件管理

hexo会将md文件处理成可以静态访问的web页面，并自动推送到github。但通过hexo编写的.md文件本身并没有做版本管理，为了方便在不同的机器上操作，可以再建一个仓库来管理。
